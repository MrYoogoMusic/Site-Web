📋 PROCHAINES ÉTAPES

  1. Configurer les secrets GitHub
     → Settings > Environments > staging/production
     → Ajouter tous les secrets listés dans README-CICD.md

  2. Protéger les branches
     → Settings > Branches > Branch protection rules
     → develop: require PR + status checks
     → main: require PR + approvals + status checks

  3. Créer les branches
     git checkout -b develop
     git push origin develop
     git checkout -b main
     git push origin main

  4. Premier déploiement staging
     git checkout develop
     git add .
     git commit -m "ci: add professional CI/CD setup"
     git push origin develop
     → Déploiement staging s'exécute automatiquement

  5. Déploiement production
     → Créer PR de develop vers main
     → Review et merge
     → Déploiement production automatique
