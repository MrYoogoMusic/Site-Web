1️⃣  CONFIGURATION GITHUB (15 min)
───────────────────────────────────
   a) Settings > Environments
      → Créer: staging
      → Créer: production

   b) Pour CHAQUE environment, ajouter 11 secrets:
      SERVER_HOST           (IP ou domaine du serveur)
      SERVER_USER           (utilisateur SSH)
      SERVER_SSH_KEY        (clé privée SSH complète)
      APP_PATH              (/home/user/rockzone)
      MONGO_URL             (mongodb://mongodb:27017)
      DB_NAME               (rockzone)
      MONGO_ROOT_USER       (admin)
      MONGO_ROOT_PASSWORD   (votre-mot-de-passe)
      REACT_APP_BACKEND_URL (https://votre-domaine.com)
      STAGING_URL           (staging uniquement)
      PRODUCTION_URL        (production uniquement)

   c) Générer clé SSH:
      ssh-keygen -t ed25519 -C "deploy@rockzone" -f ~/.ssh/rockzone_deploy
      ssh-copy-id -i ~/.ssh/rockzone_deploy.pub user@server
      cat ~/.ssh/rockzone_deploy  # Copier dans SERVER_SSH_KEY

   d) Settings > Branches > Branch protection rules
      develop: ✅ Require PR + status checks
      main:    ✅ Require PR + approvals + status checks

2️⃣  PRÉPARER SERVEUR (10 min)
──────────────────────────────
   Sur CHAQUE serveur (staging & production):

   # Installer Docker
   curl -fsSL https://get.docker.com -o get-docker.sh
   sudo sh get-docker.sh

   # Installer Docker Compose
   sudo apt-get install docker-compose-plugin

   # Vérifier
   docker --version
   docker compose version

3️⃣  PREMIER DÉPLOIEMENT (5 min)
────────────────────────────────
   # Créer branche develop et push
   git checkout -b develop
   git add .
   git commit -m "ci: add professional CI/CD setup"
   git push origin develop

   ✨ Le workflow GitHub Actions démarre automatiquement !
   → Build images Docker
   → Push sur GHCR
   → Déploie sur staging
   → Health check automatique

4️⃣  VÉRIFIER STAGING
─────────────────────
   # Sur GitHub
   Actions > Deploy to Staging > Voir les logs

   # Sur le serveur staging
   ssh user@staging-server
   cd /home/user/rockzone
   docker compose ps  # Tous les services doivent être "healthy"
   docker compose logs -f

   # Tester l'application
   curl https://staging.votre-domaine.com/api/
   curl https://staging.votre-domaine.com/

5️⃣  DÉPLOIEMENT PRODUCTION
───────────────────────────
   # Créer PR de develop vers main
   git checkout -b main
   git push origin main

   # Sur GitHub: créer Pull Request develop → main
   # Review + approve + merge

   ✨ Déploiement production automatique !
